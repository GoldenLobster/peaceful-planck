phase 0 - architecture and planning

You are a senior mobile architect.

I am building an iOS app called ytmUltimate.

Goal:
Create a native YouTube Music clone with:
- No advertisements
- Full YouTube Music-like functionality
- Background audio playback
- Lock screen controls
- Playlists
- Search
- Library
- Recommendations
- Queue management
- Album/artist pages
- Offline caching

Technical constraints:
- Flutter/Dart should be used for as much of the app as possible.
- The app should be fully native on iOS.
- JavaScriptCore will run youtubei.js.
- Swift should only be used where Flutter cannot access required iOS APIs.
- AVFoundation should handle audio playback.

Design the complete architecture:
- Flutter layer responsibilities
- Swift native layer responsibilities
- JavaScriptCore runtime responsibilities
- Communication interfaces
- Data flow
- State management approach
- Folder structure
- Long-term scalability concerns

Do not write code yet.
Produce a professional engineering design document.

Create the initial Flutter project structure for ytmUltimate.

phase 2 - native ios bridge foundation

Implement the Flutter-to-iOS communication architecture for ytmUltimate.

The app needs native iOS features:
- JavaScriptCore execution
- AVAudioSession management
- Background playback
- Lock screen controls
- Remote commands

Design a Swift bridge using:
- Flutter MethodChannels or EventChannels

- Async communication

Create:
- Swift classes
- Dart wrapper classes
- Communication protocol

Keep Swift minimal. Flutter should remain the main application layer.

Build the JavaScriptCore runtime environment for ytmUltimate.

Goal:
Run youtubei.js inside JavaScriptCore on iOS.

Requirements:
- Load bundled JavaScript files
- Create JSContext
- Expose native functions
- Handle JS errors
- Async communication

Implement missing runtime APIs:
- fetch
- Response
- Headers
- EventTarget
- CustomEvent
- TextEncoder
- TextDecoder
- URL
- ReadableStream if required

phase 4 - fetch implementation

Implement a fetch API compatible with youtubei.js inside JavaScriptCore.

Requirements:
- Use iOS URLSession internally
- Return spec-compatible Response objects
- Support:
  - status codes
  - headers
  - JSON responses
  - text responses
  - ArrayBuffer
  - ReadableStream where possible

The implementation must satisfy youtubei.js requirements.

Design the Swift-JavaScript boundary.
